# Правила работы с проектом MedSprint GEO

## Структура файлов

- Каждая страница — отдельный `.html` файл в корне проекта
- Без внешних библиотек и фреймворков: только HTML, CSS, JS
- Без систем сборки: файл должен открываться напрямую в браузере

## Дизайн

- Ориентир — `landing.html`: цвета, шрифты, карточки, анимации
- CSS-переменные из `:root` обязательны для всех новых файлов:
  ```css
  --blue:        #2388ed;
  --blue-dark:   #0b5ed7;
  --blue-deeper: #0a3f8f;
  --blue-light:  #e8f2fd;
  --blue-mid:    #c8dff8;
  --navy:        #0c1f4a;
  --text:        #1e293b;
  --text-muted:  #5a6a85;
  --border:      #dce8f8;
  --white:       #ffffff;
  ```
- Стиль элементов: скруглённые углы, тени, плавные hover-эффекты
- Не добавлять ничего визуально чужеродного — сайт должен смотреться как единое целое

## Иконки и иллюстрации

- Иконки — только inline SVG или Unicode-символы
- Внешние иконочные шрифты (Font Awesome и подобные) не использовать
- Фотографии и растровые изображения подключать тегом `<img>` с атрибутом `alt`

## Адаптивность

- Вёрстка работает корректно на экранах от 320px
- Брейкпоинты: 900px (планшет), 640px (мобильный)
- Типографика через `clamp()` — размеры шрифтов масштабируются плавно
- На мобильном: одна колонка, комфортные отступы (16px)

## Навигация между страницами

- На каждой странице — тот же navbar, что в `landing.html`
- Логотип в navbar всегда ведёт на `landing.html`
- На внутренних страницах (кейс, статья) — хлебные крошки под navbar
- Footer одинаковый на всех страницах

## SEO и мета-теги

Каждая страница должна содержать в `<head>`:
```html
<title>Заголовок страницы — МедСпринт</title>
<meta name="description" content="Описание страницы, 120–160 символов" />
<meta property="og:title" content="Заголовок страницы" />
<meta property="og:description" content="Описание для соцсетей" />
<meta property="og:image" content="URL картинки для превью" />
<link rel="canonical" href="URL страницы" />
<link rel="icon" href="favicon.ico" />
```

## Аналитика

- Скрипт Яндекс.Метрики (или другой — уточняется) добавляется в `<head>` каждой страницы
- Без аналитики страницы не публикуются

## Текст

- Простой и дружеский язык — без канцелярита и корпоративных штампов
- Обращение к читателю на «вы» (вежливое, но живое)
- Короткие предложения, конкретные формулировки

## Рабочий процесс

- Перед созданием нового файла — описать план: что будет на странице, из каких секций состоит, какова цель
- Не создавать файлы без явного согласования структуры
- Изменения в существующих файлах — только после прочтения актуального кода
